## 二、问题诊断

### 问题 1: 会话/信标/主机 — 三表分离造成重复

**现状:** Session、Beacon、Host 三个独立页面，各有独立路由和数据表。

**数据重复:**
- Session 页: `[ID, Hostname, OS, User, IP, Transport, PID, Status]`
- Beacon 页: `[ID, Hostname, OS, User, IP, Interval, LastCheckin, Status]`
- Host 页: `[Hostname, OS, Sessions, Beacons, IOC]`

三张表共享 Hostname/OS/User/IP 等核心字段。Host 页就是 Session+Beacon 按 Hostname GROUP BY 的聚合视图。

**核心矛盾:** 用户在渗透时关心的是"这台机器上有什么"，而不是"这款 implant 用了什么协议"。Host 才是操作员的心理模型，Session/Beacon 只是技术细节。

**建议:** → 见第三章方案 A

### 问题 2: 通信管理 — 4 个子页可精简为 2 个

**现状:** C2 配置、证书管理、监听器、流量编码四个页面均有后端 API 支持，功能完整。

**可优化点:**
- 证书管理和流量编码都可以作为 C2 配置页的子 Tab，不需要独立路由
- 证书管理 181 行、流量编码 199 行，体量小，合并后不影响可维护性
- 监听器独立保留（操作频率高）

**建议:** → 见第三章方案 B

### 问题 3: 行动记录 — 金丝雀/监控可合并

**现状:** 4 个子页，其中:
- 事件记录 + 命令记录 → 都是审计日志，共用一个思维模型（时间线+过滤）
- 金丝雀 (98行) + 系统监控 (302行) → 都是 Watchtower 的两种视图

**建议:** 事件+命令合并为"操作审计"，金丝雀合并到系统监控的子 Tab。

### 问题 4: AI Chat 双入口

**现状:** AI Chat 在侧栏有独立页面（含对话列表侧栏），同时右下角有 FloatingAI 浮动窗。两个入口调用同一套 API，但 UI 完全不同。

**问题:** 
- 浮动窗开了对话，切到 AI Chat 页面看不到这条记录
- AI Chat 页面有对话列表（localStorage），浮动窗没有
- 两个地方的对话历史不互通

**建议:** 统一为一个入口。详见方案。

### 问题 5: 文件管理不应是顶级路由

**现状:** 文件管理 `/implants/filebrowser` 需要先选择 session 才能用，但它独立在植入管理子菜单中，没有 session 上下文。

**建议:** 文件管理应该是 Session 详情/右键菜单的内嵌功能，不是独立页面。

### 问题 6: P2P 级联与代理管理重叠

**现状:** 代理管理页是 SOCKS/端口转发/反向转发的状态面板。P2P 级联页是 Pivot 拓扑图。两者用同一批代理 API。

**建议:** 合并为一个"隧道与拓扑"，Tab 切换代理状态和图。

### 问题 7: Payload 生成 + 构建器分离

**现状:** Payload 生成 651 行（主力），构建器 106 行（几乎空壳）。

**建议:** 构建器就是生成历史列表，可以直接作为 Payload 页的一个 Tab。

## 三、推荐方案

### 方案概述: 9 菜单 → 7 菜单，27 页 → 18 页

```
指挥中心 (dashboard)
├── 仪表盘

目标管理 (targets)           ← 重命名自"植入管理"，以主机为中心
├── 主机列表                  ← 合并 Session/Beacon/Host 为一张主机视图
│   └── [Tab] 活跃 Session
│   └── [Tab] Beacon
│   └── [Tab] IOC 标记
├── 拓扑视图                  ← 保留，右键菜单保持
└── 终端                      ← 保留（从 Session 行点击进入）

载荷工厂 (payloads)           ← 原"木马生成"，合并子页
├── 生成器                    ← Payload生成 (合并 Shellcode)
│   └── [Tab] 构建历史        ← 原"构建器"合并进来

隧道网络 (tunnels)            ← 合并"隧道代理"+"P2P级联"
├── 代理面板                  ← 原"代理管理"
└── 网络拓扑                  ← 原"P2P 级联"

通信配置 (comm)               ← 精简为 2 页
├── 监听器                    ← 保留（唯一有后端的）
└── C2 配置                   ← 合并"证书管理"+"流量编码"为子Tab

武器库 (weapons)             ← 保留
├── 扩展管理
└── 网站托管

数据收集 (loot)              ← 保留
├── 凭据管理
└── 破解站

操作审计 (audit)             ← 合并"行动记录"
├── 审计日志                  ← 合并"事件记录"+"命令记录"
└── 系统监控                  ← 合并"金丝雀"为子Tab

AI (ai)                      ← 保留，简化
└── AI 配置                   ← AI Chat 不再独立菜单（浮动窗就是入口）
```

### 详细改动清单

#### 改动 1: 植入管理 → 目标管理（$P0 最高优先级）

**当前:** 6 个子页，3 张独立数据表
**改为:** 3 个子页，1 张主机视图

**主机列表页（新 `targets/index.vue`）:**
```
┌─────────────────────────────────────────────┐
│ 目标管理                     [+刷新] [搜索]  │
├─────────────────────────────────────────────┤
│ [全部] [Session ▴3] [Beacon ▴2] [IOC]       │ ← Segment 标签
├─────────────────────────────────────────────┤
│ 主机名        OS      在线  Session  Beacon  │
│ DC01          Win19     ●  3        1        │ ← 展开行
│   ├─ session-1  admin   192.168.1.10  mTLS  │
│   ├─ session-2  SYSTEM  192.168.1.10  mTLS  │
│   └─ beacon-1   admin   192.168.1.10  HTTPS │
│ WS01          Win10     ●  1        0        │
│ DB01          Ubuntu    ○  0        1        │
└─────────────────────────────────────────────┘
```

- 主机为一级行，可展开查看关联的 Session/Beacon
- 右键菜单直接从主机行触发（自动选活跃 Session）
- IOC 标签页显示有 IOC 标记的主机
- **删除:** `implants_sessions`, `implants_beacons`, `implants_hosts` 三个独立页面

#### 改动 2: 通信管理精简（$P1）

**当前:** C2 配置 / 证书管理 / 监听器 / 流量编码
**改为:** 监听器 / C2 配置（证书+流量编码作为子 Tab）

- 删证书管理页（181 行），合并到 C2 配置页的 Tab 里
- 删流量编码页（199 行），合并到 C2 配置页的 Tab 里
- C2 配置页改为三 Tab: `[C2 配置] [证书] [编码器]`

#### 改动 3: 操作审计合并（$P1）

**当前:** 事件记录 / 命令记录 / 金丝雀 / 系统监控
**改为:** 审计日志 / 系统监控

- 审计日志页: 左边事件列表 + 右边命令列表，同一时间线
- 系统监控页: Tab `[Watchtower] [金丝雀]`
- **删除:** `operations_events`, `operations_commands`, `operations_canary` 三个独立页面

#### 改动 4: AI Chat 去重（$P2）

**当前:** AI Chat 独立页 + FloatingAI 浮动窗
**改为:** 仅保留 FloatingAI，AI Chat 页改为"对话记录"（只读）

- 或者反过来: 保留 AI Chat 页为主入口，FloatingAI 改为快捷打开 AI Chat 页的按钮
- **推荐方案:** 浮动窗里加一个"展开到全屏"按钮，点击跳转到 `/ai/chat`，两个入口共享同一套对话历史（改为后端存储而非 localStorage）

#### 改动 5: 文件管理内嵌（$P2）

- 删独立文件管理路由
- 改为 Session 详情弹窗里的一个 Tab `[详情] [文件] [进程] [网络]`
- 会话右键菜单加"文件管理"

#### 改动 6: Payload 合并（$P2）

- 删构建器独立页
- Payload 生成页加第二个 Tab: `[生成] [历史构建]`

#### 改动 7: 隧道合并（$P2）

- P2P 级联合并到代理管理页的第二个 Tab
- `[代理状态] [网络拓扑]`（拓扑图复用 P2P 页的 vis-network）

## 五、不改动的部分

以下模块保持现状，不纳入本轮重划：

| 模块 | 理由 |
|------|------|
| 指挥中心 | 仪表盘已稳定，统计维度合理 |
| 武器中心 | 扩展管理 + 网站托管两个子页功能差异大，合并无意义 |
| 数据收集 | 凭据管理 + 破解站职责清晰，后端 API 完备 |
| AI 配置 | 配置页面功能单一，不需要动 |

## 六、风险与回滚

1. **三表合一** 风险最高 — Session/Beacon/Host 的后端 API 是独立路由，前端合并后需要保证数据拉取不重复（只调 3 个 API 各一次而非多次）
2. 保留旧页面为隐藏路由（`hideInMenu: true`），出问题可随时切回
3. 每完成一个优先级就推送，不积压
